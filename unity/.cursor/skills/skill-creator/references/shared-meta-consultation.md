# Shared Meta Consultation

Use this reference when generating meta skills that plan, route, or curate context.

## Consultation Order
1. Memory/history source - check prior outcomes and avoid duplicated work.
2. Dependency analysis - identify impacted files, systems, and conflict risk.
3. Version-control context - identify safe checkpoints and rollback options.
4. Testing coordination - include right-sized validation for task risk.

## Usage Rules
- Consult only what is relevant to the request.
- Keep summaries concise and actionable.
- Include consultation output only when it changes planning decisions.
- Do not pretend consultation happened if no source is available.

## Planning Constraints
- Prefer minimal skill sets and shortest safe plan.
- Parallelize only when tracks are independent.
- Mark ambiguous or high-risk plans as confirmation-required.
